Java ha cambiado: ¿Por qué tú no?

Java ha evolucionado mucho desde Java 8, pero muchos equipos siguen escribiendo código como si aún estuviéramos en 2014. ¿Por qué? Porque aunque el lenguaje ha incorporado características modernas (lambdas, Stream API, Optional, records, sealed classes, etc.), muchos desarrolladores siguen aferrados a patrones anticuados o, peor aún, adoptan malas prácticas con las nuevas funcionalidades.

En esta charla, analizaremos ejemplos reales de código en producción que te harán cuestionar tus decisiones diarias.
Aprenderemos a identificar y evitar errores comunes, y a escribir Java de manera más eficiente, legible y mantenible, aprovechando al máximo las capacidades actuales del lenguaje.
